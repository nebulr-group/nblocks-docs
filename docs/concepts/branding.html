Branding

Flexible branding has been a top priority for us and Nblocks lets you apply your own brand to your app. This applies to all aspects like colors, logos, wordings, emailing etc. Learn how to setup your app with your own custom branding.

Branding can be customized in two ways:

  • Frontend plugins styling can be overridden with your own variables for colors, fonts, settings and translations.
  • App variables like App name, logo, email sender alias etc. that Nblocks will use in all interactions with your users. This data is stored on your app model in the NBlocks platform.

Frontend plugins​

Our frontend plugins come prebaked with colors, fonts and other default settings which can easily be changed to match your requirements.

Colors and Fonts​

  • React

Fonts​

As with colors the fonts can easily be changed using global CSS variables.

src/theme/variables.scss
:root {
  /* Fonts */
  --font-body: Inter, san-serif;
  --font-paragraph: Inter, san-serif;
  --font-heading: Inter, san-serif;
}

App's Branding Configuration​

Using our backend plugins or not, Nblocks have already stored an App model which is up to you to change. Your app model holds some variables that are central to how content like onboarding material from Nblocks looks like.

VariableDescription
nameYour app name
logoA public url to your logo
emailSenderNameThis will be the sender name in all Nblocks emails
emailSenderEmailThis will be the sender email in all Nblocks emails. You'll have to verify this email address before being able to use it
websiteUrlA public url to a landing page. Used in email footers etc.
privacyPolicyUrlA public url to a privacy policy. Used in email footers etc.
termsOfServiceUrlA public url to a terms of service. Used in email footers etc.
info

Changing the emailSenderName will send a verification email to that email address. Only after you've verified that you're the rightful owner of the email address will Nblocks use it.

Updating your app model

You can update your app model with the built in Nblocks Api client, Postman, Curl or any HTTP POST tool. Below we use the built in Nblocks Api client that can be obtained via the plugin.

  • NestJS
constructor(
    private readonly clientService: ClientService <--- Inject the ClientService
  ) {}
...
// Set branding
await this.clientService.client.updateApp({
    "name": "My App",
    "logo": "https://www.example.com/logo.png",
    "emailSenderName": "My app [Notification]",
    "emailSenderEmail": "notifications@example.com",
    "websiteUrl": "https://www.example.com",
    "privacyPolicyUrl": "https://www.example.com/privacy-policy",
    "termsOfServiceUrl": "https://www.example.com/terms-of-service"
});
