Quickstart

Overview​

The following guide will get you up and running as quickly as possible by covering the following 3 steps:

  1. Install the Nblocks plugins for both Backend and Frontend.
  2. Run our provided scripts to bootstrap your project with an initial scaffolding and other dependencies. During this phase you'll be able to signup directly.
  3. Testing the result and doing a small branding excercise 😉
note

This quickstart requires you to have npm installed.

Lets go! 🚀

Getting Started​

In this exercise we'll install Nblocks for both a frontent and a backend project to leverage the full plug-n-play experience and prebaked feature set. At this moment we support React for the frontend project and NestJS for the backend.

Backend​

Start with installing the backend! Install our plugin containing the necessary endpoints to power the Nblocks loaded frontend.

During the backend setup; the scripts will let you choose to signup for your own app profile in the Nblocks world.

Don't have a project yet?

If you don't have a project yet you can create a new skeleton project for NestJS.
nest new my-backend. Requires @nestjs/cli.

Or you can clone a working example project with this plugin installed from our Github 😉

  • NestJS
note

We support both NestJS 7 & 8.

  1. Step into your NestJS project root and run the scaffolding script to install our nblocks-nestjs plugin and all it's dependencies..
npx @nebulr-group/nblocks-plugin-tool setup-nestjs
  1. Import the NBlocksModule in your app.module.ts
src/app.module.ts
...
// Import the NBlocksModule
import { NBlocksModule } from '@nebulr-group/nblocks-nestjs';

@Module({
  imports: [
    ...
    // Add the NBlocksModule in module imports 
    NBlocksModule
    ...
  ],
})
export class AppModule {}
  1. Enable CORS in your main.ts since the Frontend project will need that to query data.
src/main.ts
...
async function bootstrap() {
  ...
  // Enable CORS Add this to enable CORS
  app.enableCors();
  ...
  await app.listen(3000);
}
...
  1. Launch your Backend. 
npm run start:dev

Frontend​

It's time for the Nblocks plugin to supercharge your frontend.

Don't have a project yet?

If you don't have a project yet you can create a new skeleton project for React.
npx create-react-app my-frontend --template typescript.

Or you can clone a working example project with this plugin installed from our Github 😉

  • React
note

We support React 18.

  1. Step into your React project root and run the scaffolding script to install our nblocks-react plugin and all it's dependencies.
npx @nebulr-group/nblocks-plugin-tool setup-react
  1. Import the NblocksProvider and surround your top most App component in App.tsx.
src/App.tsx
import { NblocksProvider, InstallationCompleteComponent } from "@nebulr-group/nblocks-react";
function App() {
  return (
    <NblocksProvider config={{ devMode: true }}>
      {/* Optional success component */}
      <InstallationCompleteComponent/> 
    </NblocksProvider>
  );
}

export default App;
Whats Dev mode?

Setting devMode: true will configure the plugin with some sensible defaults. More on this later.

  1. Launch your Frontend.
PORT=8080 npm start
Why port 8080?

Both React and the backend are using port 3000 by default. Therefore we start React on another port.

Done! Everything is installed and set!​

You are now set up, supercharged and ready to power your app with all that we have to offer. Enjoy prebuilt frontend views and full functionality for:

-> Tenancy
-> Register & onboarding
-> Login
-> Access control
-> User management
-> Checkout and billing

Let's try it!​

Create your first user and login​

  1. Try access your app by navigating to http://localhost:8080 and you should see a login screen.
  2. Click the Create new account button.
  3. Signup with a new user.
  4. Go to your email inbox, wait for the onboarding email to arrive and follow the instructions to set a password.
  5. Login with your new credentials.
  6. You're now logged in and your session is kept between page reloads, congrats!

Here's a list of the build in views included in the frontend.

  • http://localhost:8080/setup/config
  • http://localhost:8080/setup/signup
  • http://localhost:8080/auth/login
  • http://localhost:8080/auth/reset-password
  • http://localhost:8080/auth/set-password/token
  • http://localhost:8080/auth/mfa/required
  • http://localhost:8080/auth/mfa/recover
  • http://localhost:8080/auth/mfa/setup
  • http://localhost:8080/user/list
  • http://localhost:8080/auth/logout

Try adding a second user​

No fun without some friends 🤙. You're going to use our user management view to add another user!

Let's proceed! 👨‍💻

  1. If you're not logged in, login again.
  2. Navigate your browser to /user/list as in http://localhost:8080/user/list to see the User management screen.
  3. Click add user, enter an alternative email address to yourself and click invite.
  4. Check your inbox for a new onboarding email and click the link to start and complete the onboarding process. (This is going to take you through the same step you just did yourself in the previous excercise.)
  5. Login with your new user.
  6. Navigate to http://localhost:8080/user/list and you should se both your users popping up.

Mission completed! 🥊
