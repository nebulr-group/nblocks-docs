i18n & Translations

Internationalisation (i18n) support is a given requirement for any serious app project. The Nblocks platform uses i18n to generate onboarding content and other communication material in different languages for your users. What default language to use is determined by the locale property of your tenants. This is something you (programmatically) or the tenant owner can change anytime.

Frontendâ€‹

Our frontend plugins come with i18n support and during the installation process some translation files were put into your project. The Nblocks views will load these files over HTTP. Feel free to change these text entries to your own standards. You can add more languages by adding a new file named nblocks_${LANG}.json, where LANG should be lowercase in ISO_639-1 format and using the same translation keys as the other files.

The current language is automatically resolved for the logged in user. If you want to allow the user to change language via the built in frontend plugin views you need to configure this for the plugin.

  • Ionic
  • Flutter

Use the forRoot() method on the NblocksIonicModule to customize the languages.

src/app/app.module.ts
/* Enable both English and Swedish translations */
NblocksIonicModule.forRoot({
  languages: ['en','sv'],
})

Backendâ€‹

Any request that has been authorized through Nblocks will contain the tenant context and ultimately a locale property. You can use this property to set the language of the i18n plugin and render your own translations.

Since our backend plugins do not use translations we don't provide any examples. Feel free to use any tool you like.

Supported languages in Nblocks platformâ€‹

Even if you're supporting different languages in your app, the Nblocks platform (responsible for emailing, onboarding etc.) supports these languages at the time. Nblocks will fallback to English if an unknown language is present.

ISO_639-1Language
enEnglish (Default for new tenants)
svSwedish

Updated 2022-01-01

Contact us if you want to request support for a new language. We'd be happy to put that into the roadmap.
