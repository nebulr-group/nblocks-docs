Using the client to query your first data

Introâ€‹

  1. In this example, you'll code a small backend controller which will be responsible to render back a list of all your tenant workspaces.
  2. We'll make use of the built in API client to query this data straight from the Nblocks platform.
  3. You'll be testing this and see the result by using your web browser and navigating to http://localhost:3000/nblocks-tenants.
tip

Curious about the API client and what it can do? Head over the API client article.

Let's code!â€‹

Open up your backend project where our nblocks-backend plugin is installed.

  • NestJS
  • Express.js
  1. Open the up the AppController.ts and inject ClientService as a provider from our plugin.
src/app.controller.ts
import { ClientService, TenantResponseDto } from '@nebulr-group/nblocks-nestjs';
...
constructor(
    private readonly clientService: ClientService
  ) {}
  1. Create a new controller action. Let's call it nblocks-tenants.
src/app.controller.ts
@Get('nblocks-tenants')
async nblocksTenants(): Promise<TenantResponseDto[]> {
  return this.clientService.client.tenants.list();
}
tip

TenantResponseDto response type is available to import from the plugin.

  1. Before we're done we need to make sure this new controller endpoint can actually be accessed by a browser and this is done in resourceMappings.json. Add a new line to delare that the nblocks-tenants path will allow anonymous calls.
nblocks/config/resourceMappings.json
{
  ...
  "/nblocks-tenants": "ANONYMOUS",
  ...
}
info

Changes to resourceMappings.json requires a server restart.

  1. That's all you needed to do. Restart the backend and head over to the result!
